# 第十一章 指针和数组的关系

## 本章要点

在此之前，我们分别学习了数组（第三章）和指针（第七章）。数组是一片连续内存，用下标访问；指针存地址，用 `*` 间接访问。但这两者放在一起时，一些耐人寻味的现象出现了——`malloc` 返回的指针居然能用下标访问？数组名在多数场合表现得像指针？

本章把指针与数组的关系彻底梳理清楚。具体涵盖以下内容：

- 指针与数组的等价性——为什么 `p[i]` 和 `*(p + i)` 是一回事
- 指针与数组的区别——数组名不是变量、`sizeof` 的差异、赋值限制

理解了这些，你便能在两种表示之间自如切换，也为下一章理解生命周期中内存行为打下基础。

---

## 一、指针与数组的等价性

### 1. 一个令人困惑的事实：指针居然能当数组用

在第 7 章，我们学习了 `malloc`——向堆申请一块内存，返回的指针可以通过**下标**来访问，如同使用一个数组一般：

```c
int *p = malloc(5 * sizeof(int));
p[0] = 10;
p[1] = 20;
// ...
printf("%d\n", p[2]);  // 完全像数组！
```

而在第 1 章学习数组时，我们已知道数组名代表数组的首地址：

```c
int arr[5] = {1, 2, 3, 4, 5};
printf("%p\n", (void *)arr);    // 输出一个地址
printf("%p\n", (void *)&arr[0]); // 和上面完全一样
```

于是，一个自然的困惑便浮现了出来：指针和数组，到底是不是同一个东西？为什么指针可以用下标，数组名却又是一个地址？带着这两个问题，我们进入本章的核心内容。

---

### 2. 数组名——多数场景会"退化"为指针

当你写下 `int arr[5];` 时，`arr` 这个名字在本质上代表的是一个**数组对象**——它拥有确定的长度和类型，占据着一块连续的栈空间。然而，在绝大多数表达式中，当数组名出现时，编译器会自动将其转换为**数组首元素的地址**。这一行为通常被称为数组名到指针的"退化"。

为了便于初学时理解，你可以先把 `arr` 看作一个"固定不变的首元素地址"。但更准确的表述是：`arr` 并不是一个指针变量，它是数组的名字，只是在大多数使用场景下会**退化**成指向首元素的指针。下面的代码可以验证这一点：

```c
int arr[5] = {10, 20, 30, 40, 50};

// arr 的值就是 &arr[0]
printf("arr = %p\n", (void *)arr);
printf("&arr[0] = %p\n", (void *)&arr[0]);
```

两行通常打印相同的地址表示。但数组名不是“指针常量”；它标识一个数组对象，并且数组表达式在多数场景会转换为首元素指针。数组对象整体不是可修改左值，因此不能像指针变量那样重新赋值：

```c
int arr[5];
int *p = arr;     // 正确：把数组首地址赋给指针
arr = p;          // 错误！数组名不能放在等号左边被赋值
```

---

### 3. 指针算术运算——为什么 `arr[i]` 和 `*(arr + i)` 等价

既然数组名在表达式中会退化为首元素的地址，那么"第 i 个元素"的地址显然可以通过首地址加上一个偏移量来算出：

```
首地址 + 偏移量 → 第 i 个元素的地址
```

C 语言允许对指针进行**加法运算**，并且这种加法会**自动根据指针所指向的类型进行缩放**。例如，`int *p` 加 1，地址值实际增加的是 `sizeof(int)` 个字节（通常是 4 字节），正好跳跃到下一个 `int`。这项机制使得通过指针遍历数组元素变得极其自然。

因此，以下两种写法的含义完全相同：

```c
int arr[5] = {10, 20, 30, 40, 50};

// 下面两行完全等价
printf("%d\n", arr[2]);       // 输出 30
printf("%d\n", *(arr + 2));   // 输出 30
```

**`arr[i]` 的本质就是 `*(arr + i)`**。这不是巧合，而是 C 语言标准明确规定的语义——编译器在见到 `arr[i]` 时，内部会将它翻译为 `*(arr + i)`，然后再去访问对应的内存地址。反过来，如果 `i` 是一个指针而 `arr` 是一个整数偏移量，`i[arr]` 在语法上也是合法的（尽管这种写法极不推荐用于实际代码）。

现在我们就彻底明白了：为什么用 `malloc` 返回的指针也能写 `p[i]`？因为 `p` 是一个指针，`p[i]` 被翻译成 `*(p + i)`，而 `p + i` 正是第 i 个元素的实际地址——这一切的运算逻辑与数组名参与下标访问时完全一致。

---

### 4. 用指针遍历数组

基于上面的等价关系，遍历数组可以有多种写法。下面以输出数组所有元素为例，展示三种常见的方式：

```c
#include <stdio.h>

int main(void)
{
    int arr[5] = {10, 20, 30, 40, 50};

    // 方法1：下标法（最常用）
    printf("下标法：");
    for (int i = 0; i < 5; i++)
    {
        printf("%d ", arr[i]);
    }
    printf("\n");

    // 方法2：指针偏移法（用数组名）
    printf("指针偏移法：");
    for (int i = 0; i < 5; i++)
    {
        printf("%d ", *(arr + i));
    }
    printf("\n");

    // 方法3：移动指针法（用独立指针变量）
    printf("移动指针法：");
    int *p = arr;                // p 指向数组首元素
    for (int i = 0; i < 5; i++)
    {
        printf("%d ", *p);
        p++;                      // p 移动到下一个元素
    }
    printf("\n");

    return 0;
}
```

三种写法的输出完全相同：

```
下标法：10 20 30 40 50
指针偏移法：10 20 30 40 50
移动指针法：10 20 30 40 50
```

从可读性的角度而言，三种访问方式各有适用场景：

- **下标法**：最直观——它明确表达了"访问第几个元素"的语义
- **指针偏移法**：更贴近底层运算的真相，有助于理解编译器生成代码的方式
- **移动指针法**：在字符串处理、链表遍历等算法中非常常见，值得熟练掌握

以上讨论聚焦于指针与数组"可以互换使用"的那一面。然而，等价并不意味着等同——在可互换的表象之下，数组和指针有着若干本质性的差异，这正是下一节要探讨的内容。

---

## 二、指针与数组的区别

上一节我们看到了数组和指针在表达式层面的相似性：`arr[i]` 与 `*(arr + i)` 等价，数组名在多数场合会退化为首元素地址。但如果由此就认为数组名就是一个指针变量，那就走入了一个常见的误区。这一节将梳理数组与指针之间的三个关键区别，帮助你在相似的表象之下建立起清晰的边界。

### 1. `sizeof` 的结果不同

```c
int arr[5] = {1, 2, 3, 4, 5};
int *p = arr;

printf("sizeof(arr) = %d\n", (int)sizeof(arr));  // 20（整个数组的字节数）
printf("sizeof(p)   = %d\n", (int)sizeof(p));    // 8（指针本身的字节数，64位系统）
```

`sizeof` 作用于数组名和指针时，结果完全不同：

- **`sizeof(arr)`**：得到**整个数组**占用的内存大小——`5 * sizeof(int)`，大多数平台上为 20 字节。此时数组名不被视为指针（`sizeof` 是少数例外之一）
- **`sizeof(p)`**：得到**指针变量本身**的大小，64 位系统上通常是 8 字节，与它指向什么数据毫无关系

这个差异在将数组作为函数参数传递时尤为关键，我们将在下一章深入探讨。

### 2. 取地址的行为不同

```c
int arr[5];

printf("arr    = %p\n", (void *)arr);    // 首元素地址
printf("&arr   = %p\n", (void *)&arr);   // 整个数组的地址
// 这两个数值相同，但类型不同！
```

虽然 `arr` 和 `&arr` 打印出的地址值相同，但它们的**类型**截然不同：`arr` 在表达式中退化为 `int *`（指向首元素的指针），而 `&arr` 的类型是 `int (*)[5]`（指向整个数组的指针）。这种类型差异会影响指针算术运算——对 `arr` 加 1 会前进 1 个 `int` 的距离，而对 `&arr` 加 1 则会前进整个数组的距离（即 `5 * sizeof(int)` 字节）。

对比指针变量：

```c
int *p = arr;
printf("p   = %p\n", (void *)p);   // p 的值：arr 的首地址
printf("&p  = %p\n", (void *)&p);  // 指针变量 p 自己的地址（和 p 的值完全不同！）
```

这里 `p` 是一个独立的指针变量，它有自己的存储空间。`p` 的值是数组的首地址，而 `&p` 是 `p` 这个变量自身在内存中的地址——两者毫不相干。

### 3. 数组名不可修改

```c
int arr[5];
int *p = arr;

p++;    // 正确：p 是变量，可以改变指向
arr++;  // 错误！数组表达式不是可递增的指针变量
```

数组名标识的是数组对象，而不是一个单独存放地址、可被赋值的指针对象；这才是它不能放在赋值号左边或执行 `++` 的根本原因。指针变量则保存一个可重新赋值的指针值，可以先后指向不同数组。

这三个区别揭示了指针和数组在内存模型中的不同角色：数组是**数据的容器**，指针是**寻址的工具**。理解这一点，便不会被表面的语法相似性所迷惑。

## 三、动手练习

```c
#include <stdio.h>

int main(void)
{
    // 练习1：用三种不同方式输出数组
    int arr[6] = {1, 3, 5, 7, 9, 11};

    printf("练习1：三种遍历方式\n");
    // 方式1：下标法
    for (int i = 0; i < 6; i++) printf("%d ", arr[i]);
    printf("\n");

    // 方式2：指针偏移法
    for (int i = 0; i < 6; i++) printf("%d ", *(arr + i));
    printf("\n");

    // 方式3：移动指针法
    int *p = arr;
    for (int i = 0; i < 6; i++) printf("%d ", *p++);
    printf("\n\n");

    // 练习2：验证 sizeof 的区别
    printf("练习2：sizeof 区别\n");
    printf("sizeof(arr) = %d（整个数组）\n", (int)sizeof(arr));
    printf("sizeof(p)   = %d（指针本身）\n\n", (int)sizeof(p));

    // 练习3：指针数组和数组指针的简单演示
    int a = 10, b = 20, c = 30;

    // 指针数组：数组里存指针
    int *ptrArr[3] = {&a, &b, &c};
    printf("练习3：指针数组\n");
    for (int i = 0; i < 3; i++)
    {
        printf("ptrArr[%d] 指向的值：%d\n", i, *ptrArr[i]);
    }

    // 数组指针：指向整个数组
    int matrix[3] = {100, 200, 300};
    int (*arrPtr)[3] = &matrix;
    printf("\n数组指针：通过 (*arrPtr)[i] 访问\n");
    for (int i = 0; i < 3; i++)
    {
        printf("(*arrPtr)[%d] = %d\n", i, (*arrPtr)[i]);
    }

    return 0;
}
```
